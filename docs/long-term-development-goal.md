# AQSP 长期开发 Goal

## 目标

把 AQSP 建成沪深主板 + 创业板的可解释量化研究工作台：

- 盘前、盘中、盘后链路稳定，缺数据时明确显示缺口，不用历史结果冒充当天结果。
- 候选必须显示股票代码、股票名称、技术证据、消息证据和数据时间。
- 纸面变体记录今日持有、昨日持有、名称、成交动作和换票依据。
- 策略变体持续扩展，但按末端组合和参数签名去重；扩展不能牺牲运行速度或因果性。
- 变体矩阵由 `config/variant_grid.yaml` 版本化管理；当前 v4 生成 64 个组合，产物记录去重前后数量。
- walk-forward 负责评估和冻结，不能看到亏损后当天改参数。
- 代码从 GitHub immutable release 发布；运行数据、缓存、密钥和私有行情库留在 release 外。
- 只做研究和纸面验证，始终不自动下单。

## 验收门槛

1. 当前日期的行情、消息、盘前阶段状态和新鲜度都能追溯到实际落盘来源。
2. 每个变体都有独立策略逻辑；相同末端持仓会被标记或去重，不能生成重复变体。
   变体产物必须记录 `grid_version`、`generated_variant_count`、`variant_count` 和 `duplicate_portfolios_removed`。
3. 昨日持仓缺失时刷新失败并保留上一版，不能写入 `null` 或空的伪基线。
4. 回测使用不复权原始价格和 point-in-time 因子，不允许 look-ahead。
5. 发布后完成 health、快照契约、隔离无头页面和磁盘存储检查。

## 迭代顺序

- **数据链路**：先保证新鲜度、来源、盘前消息和阶段状态准确。
- **变体引擎**：再扩大策略家族和参数组合，保留样本门槛、冷却期和去重证据。
- **解释界面**：最后压缩首页信息密度，固定展示今日/昨日持有与换票逻辑。
- **发布闭环**：每次代码变更完成测试、secret scan、immutable release 和公网验收；失败时不覆盖旧版本。
