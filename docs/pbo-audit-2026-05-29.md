## 2. 对照 CSCV 标准（Bailey, López de Prado & Zhu 2014, §3.1）

### CSCV 标准步骤（论文 Algorithm 1）

1. 构建 T × N 矩阵 M（T 个时间期，N 个策略配置）
2. 将 M 按行分成 S 个等大的子矩阵
3. 枚举 C(S, S/2) 个组合，每个组合 c：
   - 训练集 J = c 中 S/2 个子矩阵拼接
   - 测试集 J_c = 补集 S/2 个子矩阵拼接
   - 在 J 中计算每个策略的 SR，找到最优 n*
   - 在 J_c 中计算 n* 的排名 w（0-1）
   - 计算 logit λ = ln(w / (1-w))
4. PBO = P(λ ≤ 0) = f(λ) 在 (-∞, 0] 的累积概率

### 三处关键差异

| # | CSCV 标准（论文） | 当前实现 | 严重程度 |
|---|-------------------|----------|----------|
| 1 | **需要 N 个不同策略配置**（不同参数/因子），构建 T×N 矩阵 | **只有 1 个策略**，N=1，无法比较策略间排名 | 🔴 致命 |
| 2 | **枚举 C(S, S/2) 个组合**（如 S=10 时有 252 种） | **顺序切分**，n=31 时只有 3-4 个 fold | 🔴 致命 |
| 3 | **计算 logit λ = ln(w/(1-w))**，取 P(λ ≤ 0) | **二值比较**（test_rank < best_train_rank → 1/0），无 logit | 🟠 严重 |

差异 #1 是最根本的：CSCV 的核心思想是比较"多个策略配置在样本内/外的排名一致性"。当前实现只有 1 个策略，把"时间期"当成了"策略配置"，概念上就不对。

## 3. 最小复现测试

### 测试设计

用 numpy 手工构造 5 组 returns 序列，分别喂给当前 `_calculate_pbo`：

| 测试 | 描述 | 数据 | 期望 PBO | 实测 PBO |
|------|------|------|----------|----------|
| 1 | 明显过拟合（前半高后半低） | `[0.20]*10 + [-0.10]*10` | > 0.5 | 0.50 |
| 2 | 无过拟合（全部稳定） | `[0.05]*20` | ≈ 0 | 0.00 |
| 3 | 弱过拟合 | `[0.08]*10 + [0.02]*10` | 0.3-0.5 | 0.50 |
| 4 | 随机噪声 | `rng.normal(0.03, 0.05, 20)` | 不确定 | 0.50 |
| 5 | 极端过拟合 | `[0.50]*10 + [-0.50]*10` | > 0.5 | 0.50 |

### 关键发现：~30 period 时 PBO 恒定 0.75

用 31 个 period（模拟真实 walkforward 的 period 数量）测试：

| 数据 | PBO |
|------|-----|
| 真实收益序列（31 期） | **0.75** |
| shuffled(seed=1) | **0.75** |
| shuffled(seed=2) | **0.75** |
| shuffled(seed=3) | **0.75** |
| shuffled(seed=42) | **0.75** |
| shuffled(seed=99) | 0.50 |

**5 次随机打乱中 4 次输出 0.75**。

### 结构性偏差的根因

n=31 时 `splits = 31//3 = 10`，产生 4 个 fold（[0-9], [10-19], [20-29], [30]）。

每个 fold 的二值判断 `test_rank < best_train_rank`：
- 最佳训练 period 的全局排名通常是最高或接近最高
- 测试集的平均排名通常低于最佳训练排名
- 4 个 fold 中约 3 个满足条件 → 3/4 = 0.75

**这是结构性偏差，不是真实的过拟合度量。**
